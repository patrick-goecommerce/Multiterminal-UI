You are working on a C#/.NET project. Follow these principles:

- **ASP.NET Core patterns:** use minimal APIs for simple endpoints, controllers for complex resources. Register services in `Program.cs` with `builder.Services`. Use the built-in DI container ‚Äî constructor inject interfaces, not concrete types.
- **Entity Framework Core:** use code-first migrations. Define entities as POCOs with navigation properties. Use `DbContext` with `DbSet<T>` per entity. Always use `AsNoTracking()` for read-only queries. Use `Include()` for eager loading ‚Äî never lazy loading in web apps.
- **LINQ:** prefer method syntax (`Where`, `Select`, `GroupBy`) for chained queries. Use `async` variants (`ToListAsync`, `FirstOrDefaultAsync`) for EF queries. Never call `ToList()` before applying filters ‚Äî filter in the database.
- **Async/await:** use `async Task<T>` for all I/O-bound methods. Never use `.Result` or `.Wait()` ‚Äî they cause deadlocks. Use `ValueTask<T>` for hot paths that may complete synchronously. Use `CancellationToken` in all async methods that accept it from the framework.
- **Error handling:** use middleware for global exception handling. Define domain exception types. Use `ProblemDetails` (RFC 7807) for API error responses. Use `Result<T>` pattern (custom or OneOf) for business logic errors ‚Äî reserve exceptions for unexpected failures.
- **Configuration:** use `IOptions<T>` pattern with strongly-typed settings classes. Bind from `appsettings.json` with `builder.Configuration.GetSection("Name").Bind(settings)`. Use `IOptionsMonitor<T>` for reloadable config.
- **Dependency injection:** register services with appropriate lifetimes ‚Äî `Scoped` for per-request (DbContext, repositories), `Singleton` for stateless services, `Transient` for lightweight stateless utilities. Never inject `Scoped` into `Singleton`.
- **DTOs and mapping:** separate API models from domain entities. Use records for immutable DTOs. Map with AutoMapper or manual extension methods. Use `[FromBody]`, `[FromQuery]`, `[FromRoute]` to clarify binding sources.
- **Validation:** use FluentValidation for complex rules, Data Annotations for simple property validation. Validate at the API boundary. Return `ValidationProblemDetails` for validation failures.
- **Security:** use ASP.NET Core Identity or JWT Bearer authentication. Apply `[Authorize]` at controller or endpoint level. Use policy-based authorization for complex rules. Store secrets in User Secrets (dev) or Azure Key Vault (prod), never in `appsettings.json`.
- **Testing:** use xUnit for unit tests, `WebApplicationFactory<Program>` for integration tests. Mock dependencies with Moq or NSubstitute. Use `Bogus` for test data generation. Test at the behavior level ‚Äî call endpoints, assert responses.
- **Project structure:** use Clean Architecture or Vertical Slice. Separate API, Application, Domain, and Infrastructure layers. Domain has no dependencies. Use MediatR for decoupled request handling if the project warrants it.
- **Modern C#:** use primary constructors (C# 12), collection expressions, pattern matching, `required` properties, file-scoped namespaces, global usings. Target .NET 8+ for new projects.
- **Performance:** use `IMemoryCache` or `IDistributedCache` for caching. Enable response compression. Use `System.Text.Json` source generators for high-throughput serialization. Profile with `dotnet-trace` and BenchmarkDotNet.
