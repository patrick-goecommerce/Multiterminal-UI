You are working on a Rust project. Follow these principles:

- **Ownership and borrowing:** prefer borrowing (`&T`, `&mut T`) over cloning. Clone only when ownership transfer is genuinely needed. Use `Cow<'_, str>` when a function may or may not need to allocate. Avoid `Rc`/`Arc` unless shared ownership is truly required.
- **Error handling:** use `Result<T, E>` for all fallible operations. Define domain error enums using `thiserror`. Use `anyhow::Result` in application code and binaries; use typed errors in library code. Propagate errors with `?` — never `unwrap()` in production code except when the invariant is provably guaranteed.
- **Option handling:** prefer combinators (`map`, `and_then`, `unwrap_or_default`) over `match` for simple transformations. Use `if let Some(x) = value` for single-variant matching. Never `unwrap()` an Option in production without a preceding guarantee.
- **Lifetimes:** let the compiler infer lifetimes whenever possible. Add explicit annotations only when the compiler requests them. If lifetime annotations become complex, consider restructuring to use owned data or `Arc`.
- **Async with Tokio:** use `tokio::spawn` for concurrent tasks, `tokio::select!` for racing futures, `tokio::join!` for waiting on multiple futures. Use `async fn` in traits (Rust 1.75+). Cancel tasks by dropping their `JoinHandle` or using `CancellationToken`.
- **Structs and enums:** prefer enums for state machines and variant types. Use `#[derive(Debug, Clone)]` on most types. Implement `Display` for user-facing types, `Debug` for developer-facing. Use `#[non_exhaustive]` on public enums in libraries.
- **Traits:** keep traits small and focused (1-3 methods). Use default implementations when sensible. Prefer generics (`fn foo<T: Trait>(x: T)`) for static dispatch; use `dyn Trait` only when dynamic dispatch is needed (heterogeneous collections, plugin systems).
- **Collections:** use `Vec` as the default collection. Use `HashMap`/`HashSet` for lookups. Pre-allocate with `with_capacity` when the size is known. Iterate with iterators and combinators over manual indexing.
- **Concurrency:** use `Arc<Mutex<T>>` for shared mutable state. Prefer `tokio::sync::Mutex` in async code (not `std::sync::Mutex`). Use channels (`tokio::sync::mpsc`, `flume`) for message passing. Minimize lock scope.
- **Testing:** use `#[cfg(test)] mod tests` in each file. Use `#[test]` for unit tests, `tests/` directory for integration tests. Use `assert_eq!` with descriptive messages. Create builder patterns or fixtures for complex test setup.
- **Serialization:** use `serde` with `#[derive(Serialize, Deserialize)]`. Use `#[serde(rename_all = "camelCase")]` for JSON APIs. Use `#[serde(default)]` for backward-compatible deserialization. Validate after deserializing — serde is not a validation framework.
- **Dependencies:** audit with `cargo audit`. Prefer widely-used crates with active maintenance. Pin major versions in `Cargo.toml`. Use `cargo clippy` and `cargo fmt` in CI — treat clippy warnings as errors.
- **Performance:** profile before optimizing. Use `#[inline]` sparingly — the compiler usually knows better. Prefer stack allocation over heap. Use zero-copy parsing (`&str` slices) when processing large text.
