You are working on a PHP/Laravel project. Follow these principles:

- **PHP 8+ features:** use typed properties, union types, named arguments, enums, `match` expressions, readonly properties/classes, and constructor promotion. Enable strict types with `declare(strict_types=1)` in every file.
- **Laravel conventions:** follow the directory structure conventions. Use Artisan generators (`make:model`, `make:controller`) for scaffolding. Name controllers as singular resource (`UserController`). Use RESTful resource routes.
- **Eloquent:** use relationships (`hasMany`, `belongsTo`, `belongsToMany`) over raw joins. Use eager loading (`with()`) to prevent N+1 queries. Define scopes for reusable query logic. Use accessors and mutators (attribute casting in Laravel 9+).
- **Controllers:** keep controllers focused — one resource per controller. Use form request classes for validation (`make:request`). Use resource classes for response formatting (`make:resource`). Delegate complex logic to action or service classes.
- **Validation:** use Form Request classes with `rules()` method. Define custom validation rules with `make:rule`. Return 422 with structured error messages. Validate at the boundary — never trust input in service classes.
- **Service layer:** extract business logic from controllers into action classes (`app/Actions/`) or service classes. Keep them single-responsibility. Inject dependencies through the constructor. Return typed results, not mixed arrays.
- **Error handling:** use `app/Exceptions/Handler.php` for global exception handling. Define domain exceptions extending `DomainException` or `RuntimeException`. Return consistent JSON error responses with status codes.
- **Database:** always use migrations for schema changes. Use database transactions (`DB::transaction()`) for multi-step writes. Add indexes on columns used in queries. Use Eloquent query builder over raw SQL. Use seeders and factories for test data.
- **Composer and PSR:** follow PSR-12 coding standards. Use Composer autoloading (PSR-4). Keep `composer.json` tidy with `composer normalize`. Run `composer audit` for security vulnerabilities.
- **Queues and jobs:** use Laravel queues for time-consuming tasks (emails, API calls, reports). Make jobs idempotent with `ShouldBeUnique` or custom logic. Implement `ShouldQueue` interface. Handle failed jobs with `failed()` method.
- **Testing:** use PHPUnit or Pest. Use feature tests for API endpoints with `$this->getJson()`, `$this->postJson()`. Use unit tests for services and domain logic. Use factories for test data. Use `RefreshDatabase` trait for test isolation.
- **Security:** use CSRF protection (enabled by default). Escape output with `{{ }}` in Blade (never `{!! !!}` for user data). Use prepared statements (Eloquent handles this). Hash passwords with `Hash::make()`. Use policies for authorization.
- **Configuration:** use `.env` for environment values, `config/` files for app config. Access via `config('app.name')`, never `env()` outside config files (values are cached). Use `php artisan config:cache` in production.
- **Performance:** use route caching, config caching, and view caching in production. Use Redis for sessions and cache. Chunk large queries with `chunk()` or `lazy()`. Use database query logging in development to catch N+1 issues.
