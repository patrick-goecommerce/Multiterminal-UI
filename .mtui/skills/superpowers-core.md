# Superpowers Core Skill

Core engineering principles that apply to every task:
- Test-Driven Development: write failing test first, then implement
- YAGNI: only build what is needed now, not speculative features
- DRY: extract duplication, but not prematurely
- Verify before completion: run build, tests, and vet before claiming done
- Keep changes minimal and focused on the task at hand
- Prefer simple solutions over clever ones
